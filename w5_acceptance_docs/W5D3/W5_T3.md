## 2. Error-Code Mapping (Redeem Voucher)

When verifying a voucher code, the backend may return specific error codes. We map these to user-friendly Vietnamese messages:

| Backend Error Code | Displayed Message | Tone/Color |
|---|---|---|
| `VOUCHER_CODE_USED` | Voucher này đã được sử dụng. | Warning (Yellow) |
| `VOUCHER_CODE_EXPIRED` | Voucher này đã hết hạn sử dụng. | Warning (Yellow) |
| `WRONG_PARTNER` \| `FORBIDDEN` | Voucher này không thuộc về đối tác hiện tại. | Error (Red) |
| `VOUCHER_CODE_NOT_FOUND` | Không tìm thấy mã voucher này trong hệ thống. | Error (Red) |
| `INVALID_BRANCH_SCOPE` | Voucher này không áp dụng tại chi nhánh đã chọn. | Error (Red) |
| *Other/Fallback* | (Displays `error.response.data.message` or default error) | Error Toast (Red) |

## 3. General Layout Checks
- Dashboards use CSS grid (`grid-cols-1 md:grid-cols-X`) allowing graceful degradation on mobile (stacking vertically).
- All tables are wrapped in `div.overflow-x-auto` to allow horizontal scrolling on small screens instead of cutting off text or squishing columns.
- Modals use `fixed inset-0` with `backdrop-blur-sm` and a scrollable `max-h-[90vh]` container to prevent them from becoming taller than the viewport.
